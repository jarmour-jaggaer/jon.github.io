Agent Detail — 7 agents, updated to requirements

Items marked ▲ Updated reflect corrections or additions from the requirements document.

Phase 1 — Intake & Routing (Customer-Facing)
1
Triage Agent ▲ Updated
AI Agent · Customer-Facing in JAI
Customer-facing front door operating within JAI. Conducts a dynamic digital interview using conditional logic, validates data formats in real-time, gates the hand-off until all mandatory fields are collected, and creates the Salesforce case record in the background before a human agent ever joins.
  • Maps customer input to Core > Product > Root Cause library via NLP
  • Triggers conditional question sets tailored to the identified root cause
  • Validates data formats in real-time (ID length, field type, endpoint URL format)
  • Gates hand-off until mandatory fields are collected — preserves data quality for reporting
  • Creates Salesforce case record in background upon Core/Product/Root Cause identification
  • Routes to correct Team/Tier based on root cause routing matrix
  • Provides full transcript to live agent within Salesforce console on assignment
Phase 2 — Resolution Enablement (SFDC-Embedded)
2
AI Recommendations Agent ▲ Updated
AI Agent · Salesforce Service Cloud
Embedded in the Salesforce Service Cloud console as a dedicated sidebar component. Surfaces real-time solution recommendations, relevant KB articles, and next best actions while agents are live on a case. Includes an explicit agent feedback mechanism that feeds model retraining.
  • Analyzes case context in real-time within Salesforce (description, customer history, product, issue type)
  • Recommends solutions, KB articles, and next best actions — with configurable recency weighting on historical data
  • Agents can review, edit, and one-click copy recommended text into case communication
  • Thumbs up/down + optional free-form comment feedback captured per recommendation
  • Feedback associated with specific recommendation and case for AI model training
  • Reports on recommendation adoption rate, accuracy, and impact on AHT/FCR
Phase 3 — Post-Resolution Intelligence
3
Wrap Up Agent
AI Agent · Phase 3
Consistent case closure and root cause validation. Bridges into the KB Creation workflow by generating the structured case summary that triggers article drafting.
  • Generates structured problem and resolution summary
  • Re-evaluates root cause accuracy against triage classification
  • Feeds corrections back to Triage Agent for routing improvement
  • Closure summary surfaces agent option to flag case for KB article generation
4
KB Creation Agent ▲ Updated
AI Agent · Human-Gated
Activates only when an agent explicitly flags a case for KB article generation at closure. Generates a draft formatted to pre-defined templates — then routes to a human review queue. Not fully autonomous.
  • Triggered by explicit agent flag at case closure in Salesforce — not autonomous
  • Analyzes full case interaction: description, notes, solution, transcripts, email exchanges
  • Generates draft formatted to pre-defined templates and style guidelines
  • Draft submitted to Salesforce Knowledge review queue with source case ID and flagging agent
  • Reviewers can accept, reject, edit, or send back for revision
  • Structured feedback (Good draft / Needs edits / Incorrect / Formatting) captured for model retraining
5
QA Agent ▲ Updated
AI Agent · Phase 3
Reviews and scores all case interactions against configurable rubrics across three dimensions: customer experience, solution effectiveness, and agent interaction quality. Supports multi-language processing for GCC global operations.
  • Scores against weighted, configurable rubrics with pass/fail thresholds
  • Evaluates CX (sentiment, frustration), Solution (accuracy, completeness), and Agent quality (empathy, adherence)
  • Supports multi-language processing across all GCC regions
  • Flags cases for human review based on configurable thresholds with reason surfaced
  • QA leads can override AI scores; overrides feed back into model retraining
  • Supports different QA rubric profiles per case type or agent role
6
RCA Agent
AI Agent · Phase 3
Analyzes root cause data from Salesforce to evaluate volume and complexity, producing structured recommendations for product/engineering teams and AI deflection prioritization.
  • Ingests structured root cause data from SFDC with volume and complexity metrics (ATT, agent hours, transfers, contacts per case)
  • High volume + low complexity → AI deflection recommendation
  • Low volume + high complexity → Product/Engineering enhancement recommendation
  • Configurable thresholds for volume and complexity classification
  • Generates monthly report with supporting data (volume, ATT, agent hours) per recommendation
Phase 4 — Closed Case Management ▲ New Agent
7
Closed Case Review Agent ▲ New
AI Agent · Autonomous · SFDC-Triggered
Monitors Salesforce for cases reaching "Closed - Customer Responded" status. Uses NLP to classify the customer reply as Action Required or No Action Required, then autonomously updates the case, reassigns it, or closes it with an automated confirmation email — all without human involvement unless confidence falls below 75%.
  • Triggered by SFDC status change to "Closed - Customer Responded"
  • NLP classifies customer reply: Action Required vs. No Action Required
  • Confidence threshold: below 75% flags case for manual human review
  • Action Required: reopens case, changes status to "Active in Support," reassigns to previous owner, adds case history comment
  • No Action Required: closes case, triggers automated no-reply email confirming closure and directing customer to open new case
  • Logs every action taken for audit and troubleshooting purposes
  • Reports on classification accuracy, false positive rate (re-opened then manually closed), and cases flagged for manual review
